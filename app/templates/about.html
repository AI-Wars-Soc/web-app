About CUWAIS

(Name subject to change)

The Cambridge University Warring AI Society is a new society trial run. Each holiday we will set a game (e.g: Chess) for competitors to create the best original AI to compete in. This CUWAIS website then operates as an ongoing scoreboard where you can submit your AIs to battle against other user's submissions, live! You will be able to make as many submissions as you want and see statistics on how each performs, allowing you to fine-tune your AI over the course of the competition.

Join our Discord for constant updates as the competition progresses!

Game Rules

The game to solve is called Chess960. It was invented by Bobby Fischer and is a simple variant of chess where the starting positions of all of the pieces are randomised behind the pawns (with a few constraints).

You must attempt to repeatedly return winning moves from the positions that your AI is given, without running out of time on the chess clock. As you win more games your score will increase, so when you think you have a better version of your repository make a submission so you can start earning more points!

If you run out of time, make an invalid move, use up your memory allocation or crash then you forfeit the game, so we make sure that your AI can play at least one 'healthy' game before it is released into the pool of other submissions. This means you don't have to worry about testing your AI too thoroughly before submission. Just get Coding to get Winning!

How to Compete

  • To start competing, log in using your Cambridge Email through Google. A CUWAIS account will then be created automatically for you and you will be logged in.
  • You then need to clone the Github Repository for the signatures of the methods that you need to implement.
  • Once logged in, you can make a submission, where your repository will be cloned onto CUWAIS servers and continuously paired up with other user's submissions.
  • You can then regularly check the leaderboard to see how you stack up against other competitors, or check your submission statistics to see a more detailed analysis of your AI's performance!

Requirements

  • All submissions must be written in Python and be self-contained. The VM that runs your code is not networked so you cannot rely on external websites, scripts or libraries.
  • Your program has access to:
    • Numpy, Scipy, TensorFlow, Keras, Theano, Lasagne
    • 1GB of RAM
    • 1/2 of a (~3GHz) CPU core
    • a chess clock style timeout passed as a parameter (starting at 10s)
    • full read access to your files in your repository
    (this is all subject to change).
  • You do NOT have access to a GPU, the internet, multiple cores, an FPGA, etc. Therefore:
    • Your program should be completely stateless. The game is not guaranteed to start from a usual starting position
    • You should attempt to use as close to the above footprint as possible. Your AI will be terminated if you go over the given limits
    • Do not try to break out of the sandbox provided
    • Do not try to read the opponent's code or moves pre-emptively
    • Do not process 'out of turn'
    • Do not be multi-threaded or in anyway be concurrent
    • Do not search for extra processors or processing devices
    There are plenty of safeguards in place to stop you doing this accidentally, but if you are found to have attempted to gain access to the results database, the VM host or the network you will be disqualified from all future events. If you find such a vulnerability please report it responsibly.

Advice

(To be added to as the competition progresses)

  • Start with minor improvements over the base AI. If your AI makes an invalid move you immediately concede so ensure that you completely understand the API before doing anything crazy
  • If you run out of time you immediately concede, so ensure that any searches have adjustable depths for if your game takes a while
  • The amount of processing power given to you is too small to evaluate a large neural net. Don't just throw compute at the problem - be smart!

Good Luck!
